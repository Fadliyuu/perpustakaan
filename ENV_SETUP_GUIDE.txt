================================================================================
                    PANDUAN SETUP ENVIRONMENT VARIABLES
================================================================================

FILE YANG SUDAH DIBUAT:
-----------------------
1. backend/env.txt              ‚Üí Untuk Railway (Production)
2. backend/env-local.txt        ‚Üí Untuk local development (rename jadi .env)
3. frontend/env.txt             ‚Üí Untuk local development
4. frontend/env-production.txt  ‚Üí Untuk Vercel (Production)

================================================================================
                        CARA MENGGUNAKAN
================================================================================

A. SETUP LOCAL DEVELOPMENT
---------------------------

1. BACKEND:
   - Buka file: backend/env-local.txt
   - Copy semua isinya
   - Buat file baru: backend/.env
   - Paste isinya ke backend/.env
   - Edit bagian Cloudinary dan JWT_SECRET sesuai kebutuhan Anda

2. FRONTEND:
   - Buka file: frontend/env.txt
   - Copy semua isinya
   - Buat file baru: frontend/.env
   - Paste isinya ke frontend/.env

================================================================================

B. SETUP RAILWAY (Backend Production)
--------------------------------------

1. Buka https://railway.app
2. Login ‚Üí New Project ‚Üí Deploy from GitHub
3. Pilih repository Anda
4. Settings ‚Üí Variables ‚Üí Add Variable
5. Copy-paste dari file: backend/env.txt

   VARIABLES YANG PERLU DI-SET:
   ----------------------------
   PORT=4000
   NODE_ENV=production
   FRONTEND_URL=https://your-frontend.vercel.app
   
   FIREBASE_PROJECT_ID=peminjaman-buku-223b6
   FIREBASE_PRIVATE_KEY="-----BEGIN PRIVATE KEY-----\n...\n-----END PRIVATE KEY-----\n"
   FIREBASE_CLIENT_EMAIL=firebase-adminsdk-fbsvc@peminjaman-buku-223b6.iam.gserviceaccount.com
   
   CLOUDINARY_CLOUD_NAME=your-cloud-name
   CLOUDINARY_API_KEY=your-api-key
   CLOUDINARY_API_SECRET=your-api-secret
   
   JWT_SECRET=change-this-to-random-secret-min-32-chars

6. UPDATE SETELAH FRONTEND DEPLOY:
   - Setelah frontend deploy ke Vercel, dapatkan URL-nya
   - Update FRONTEND_URL di Railway dengan URL Vercel

================================================================================

C. SETUP VERCEL (Frontend Production)
-------------------------------------

1. Buka https://vercel.com
2. Login ‚Üí Add New Project ‚Üí Import GitHub repo
3. Settings ‚Üí Environment Variables ‚Üí Add
4. Copy-paste dari file: frontend/env-production.txt

   VARIABLE YANG PERLU DI-SET:
   ---------------------------
   VITE_API_BASE_URL=https://your-backend.railway.app/api

5. UPDATE URL:
   - Setelah backend deploy ke Railway, dapatkan URL-nya
   - Update VITE_API_BASE_URL dengan URL Railway + /api

================================================================================

D. YANG PERLU DI-GANTI
-----------------------

1. CLOUDINARY CREDENTIALS:
   - Daftar di: https://cloudinary.com/users/register_free
   - Ambil credentials dari Dashboard
   - Ganti di backend/env.txt atau backend/.env:
     * CLOUDINARY_CLOUD_NAME
     * CLOUDINARY_API_KEY
     * CLOUDINARY_API_SECRET

2. JWT_SECRET:
   - Generate random string minimal 32 karakter
   - Bisa pakai: node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
   - Ganti di semua file env

3. URL PRODUCTION:
   - Setelah deploy, update:
     * FRONTEND_URL di Railway dengan URL Vercel
     * VITE_API_BASE_URL di Vercel dengan URL Railway + /api
